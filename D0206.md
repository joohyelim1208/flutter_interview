# 플러터 면접 스터디

## OOP
- 0206 : Q. 상속이란?
핵심: 코드의 재사용성, 유연화, 다형성 구현
추가하기: '메서드 함수: 특정 개체가 수행하는 동작이나 기능을 정의'

A. 상속이란 코드의 재사용성을 높이고 계층구조를 형성하는 것이라고 볼 수 있습니다.
객체지향 프로그래밍에서 상속은 부모클래스인 기존 클래스의 속성과 메서드를 새로운 클래스인 자식클래스가 그대로 물려받아 @override해서 사용하는 것 인데 extends를 사용해 물려받고, super를 사용해 부모클래스에 접근합니다.

상속을 하는 이유는 중복된 코드를 줄여 재사용성을 높이고 클래스 간 계층구조를 형성하여 다형성을 구현하는 바탕이 되기 때문입니다.

재사용성은 부모클래스의 공통된 속성과 행동을 한번만 정의해두면 수십개의 자식클래스에서 다시 코딩할 필요 없이 그대로 사용할 수 있으며, 필요한 것만 고쳐서 사용할 수 있습니다. 

코드를 예시로 들어보면, Todo App의 entity 클래스의 예시입니다.
```dart
// 부모 클래스: 공통 속성을 모아둔 설계도
class BaseEntity {
  final int id;  // 객체. 모든 데이터에 공통으로 필요한 고유 ID
  final DateTime createdAt; // 데이터가 생성된 시간을 기록할 변수

  BaseEntity({required this.id, required this.createdAt}); // 생성자. 데이터를 생성할 때 필수로 받도록 합니다
}

// 상속받은 자식 클래스: extends로 부모클래스의 기능을 물려받습니다. 
class TodoEntity extends BaseEntity {
  final String title; // 자식클래스가 가지는 고유한 속성

  TodoEntity({ // 자식클래스 생성자.
    required int id, // 부모에게 전달할 ID
    required DateTime createdAt, // 부모에게 전달할 생성시간
    required this.title, // 자신의 속성인 타이틀
  }) : super(id: id, createdAt: createdAt); // 부모 생성자 호출해서 데이터를 부모에게 전달해줌
}
```

만약 부모에게 물려받은 기능이 자식에게 맞지 않을 땐 자식클래스에서 내용을 override(덮어써서) 재정의 할 수 있습니다.
동일한 이름의 메서드로 각 객체마다 다르게 동작하는 다형성을 구현하기 위해 사용합니다.

```dart
// [1] 부모 클래스: 일반적인 엔티티
class BaseEntity {
  void printSummary() {
    print("이것은 기본 데이터 엔티티입니다.");
  }
}

// [2] 자식 클래스: Todo 전용 엔티티. 부모클래스를 상속(extends)받아 기능을 확장함
class TodoEntity extends BaseEntity {
  final String title;

  TodoEntity({required this.title}); // 자식클래스의 생성자.

  // [3] 오버라이딩 적용. 이 표시가 있으면 부모에게 똑같은 이름의 메서드가 있는지 확인해줌
  // 부모의 printSummary를 무시하고, Todo에 맞는 출력 방식으로 바꿉니다.
  @override
  void printSummary() {
    // 부모의 것을 재정의해서 아래 코드가 실행됩니다. 같은 메서드지만 자식마다 다른 결과를 만들 수 있습니다.
    print("할 일 제목: $title");
  }
}

void main() {
  BaseEntity base = BaseEntity(); // 부모타입 객체 생성
  TodoEntity todo = TodoEntity(title: "플러터 공부"); // 자식타입 객체 생성

  // 같은 이름의 메서드를 호출하지만 결과는 달라지게 됩니다.
  base.printSummary(); // 출력: 이것은 기본 데이터 엔티티입니다.
  todo.printSummary(); // 출력: 할 일 제목: 플러터 공부 (오버라이딩된 결과)
}
```
이렇게 하나의 객체나 메서드가 여러 형태를 가질 수 있는 성질을 다형성이라고 하며 코드를 유연하게 작성할 수 있도록 도와줍니다.
동일한 메서드만 호출하면 되기에 코드도 단순해지므로 유지보수도 편리하면서 기능을 확장하는데 필수적인 부분이라고 볼 수 있습니다.
